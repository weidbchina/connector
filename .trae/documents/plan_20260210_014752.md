# 数据库监控工具实施计划

根据您的需求规格说明书（V1.0），我为您制定了以下开发计划。本工具将采用前后端分离架构（Spring Boot + React），最终可打包为轻量级单体应用部署。

## 1. 技术架构选型

* **后端**: Java 17, Spring Boot 3.x

* **前端**: Vue3(用于实现IDE级SQL编辑体验)

* **元数据存储**: H2 Database (嵌入式，用于存储配置和日志，保证工具轻量级)

* **数据库连接**: Oracle JDBC Driver (ojdbc8)

* **调度框架**: Quartz (支持复杂的Cron表达式)

* **SQL解析**: JSqlParser (用于严格校验SELECT语句)

## 2. 详细实施步骤

### 第一阶段：项目初始化与基础架构

1. **项目结构搭建**

   * 创建 `backend` (Spring Boot) 和 `frontend` (React) 目录结构。

   * 配置 Maven 依赖（Web, JPA, H2, Quartz, OJDBC, JSqlParser）。
2. **核心数据模型设计**

   * `DbConnection`: 存储连接信息（密码字段使用AES加密）。

   * `MonitorTask`: 存储监测任务配置（SQL, 频率, 阈值规则）。

   * `TaskLog`: 存储执行日志和告警记录。
3. **安全组件开发**

   * 实现 `AESUtils`：用于数据库密码的加密存储与解密使用。

### 第二阶段：数据监测告警模块开发 (后端优先)

1. **多数据源管理**

   * 实现动态 `DataSource` 管理器，支持运行时创建/销毁Oracle连接池。
2. **监测任务引擎**

   * 集成 Quartz Scheduler，实现任务的动态增删改查与调度。

   * 开发 `SqlExecutor`：

     * 强制检查 SQL 类型（仅允许 SELECT）。

     * 实现超时控制（`Statement.setQueryTimeout`）。
3. **规则校验与告警**

   * 实现校验逻辑：数值阈值、正则匹配、空值检查等。

   * 开发 `AlertService`：

     * 实现防刷机制（Redis或内存缓存记录上次告警时间）。

     * 定义短信接口（提供模拟实现，预留第三方SDK接入点）。

### 第三阶段：Web SQL 云客户端开发 (前后端协同)

1. **SQL 编辑器集成 (Monaco Editor)**

   * 配置 Oracle 语法高亮。

   * 实现自定义补全 Provider (`CompletionItemProvider`)。
2. **智能补全后端服务**

   * 开发 `MetadataService`：查询 `ALL_TABLES`, `ALL_TAB_COLUMNS`。

   * 实现元数据缓存机制（Caffeine Cache），确保 <800ms 响应。
3. **SQL 执行接口**

   * 开发分页查询接口，支持 limit/offset。

   * 实现结果集 CSV 导出功能。

### 第四阶段：前端界面实现与集成

1. **连接管理页面**：连接列表、新建/测试连接（密码字段脱敏）。
2. **监测任务管理页面**：任务列表、Cron表达式生成器、日志查看器。
3. **SQL 工作台页面**：

   * 左侧：数据库/表结构树。

   * 中间：Monaco SQL 编辑器。

   * 底部：执行结果 Grid（支持排序、列宽调整）。

### 第五阶段：验证与交付

1. **功能验证**

   * 测试 DML 拦截功能。

   * 测试 10000 行数据限制。

   * 验证监控报警的防刷逻辑。
2. **打包交付**

   * 配置前端 build 产物复制到 Spring Boot `static` 目录。

   * 生成最终的可执行 JAR 包。

## 待确认事项

* 元数据存储默认使用嵌入式 H2 数据库以满足“轻量级”需求，无需额外部署数据库。
